## 🔧 Что можно улучшить

### 1. E2E Saga: 13-13 → 13-14 (cancelled) → 13-15

**Timeline:**
```
13-13: FK constraint fix → 20 passed, 36 failed
13-14: Попытка фиксить по одному → 40 passed, 15 failed
       Каждый fix добавлял complexity
       Тесты загрязнялись workarounds (if isCI...)
       → CANCELLED
13-15: Rewrite from scratch → 55 tests, 7.3s, 5/5 green
```

**Root Cause Analysis:**
1. **Тесты писались для другого контекста** — оригинальные E2E (Epic 12) для локального Keycloak/БД
2. **Отсутствие CI-first mindset** — "работает локально = заработает в CI"
3. **Sunk cost bias** — "уже написали 59 тестов, не можем выбросить"

**Урок:** Если тесты изначально не были designed для CI — проще переписать, чем адаптировать.

### 2. Keycloak UUID Sync Issue (Story 13-13)
Неожиданная проблема — Keycloak генерирует новые UUID при каждом импорте realm, что ломало FK constraint в audit_logs.

Решили через sync script, но это был сюрприз.

### 3. Grafana Alerts Migration (Story 13-11)
Prometheus alerts мигрировали, но Grafana alerts (file-based provisioning) не совместимы с centralized Grafana (UI-based). Документация создана в docs/monitoring-alerts.md.
