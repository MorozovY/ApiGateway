## 🔧 Что можно улучшить

### 1. Cross-service Consistency
**Problem:** Story 14-5 — забыли добавить `MdcContextConfig` в gateway-admin. Trace-log correlation не работал бы в половине системы.

**Root Cause:** При добавлении cross-cutting concern в один сервис легко забыть второй.

**Lesson:** Нужен checklist при добавлении configs/filters/dependencies — проверять ВСЕ сервисы.

### 2. Static Re-exports блокируют Code Splitting
**Problem:** Story 14-4 — `ApprovalsPage` статически экспортировался в `index.ts`, что полностью блокировало lazy loading. HIGH severity issue на code review.

**Root Cause:** React.lazy не может split chunk если компонент уже в main bundle через static export.

**Lesson:** При code splitting проверять что lazy components не имеют static re-exports.

### 3. Test Anti-patterns
**Problem:** Story 14-6 — `waitForTimeout`, hardcoded timeouts в E2E тестах. Всё исправлено на code review.

**Root Cause:** Привычка из локальной разработки, где timeouts "работают".

**Lesson:** Lint rules против anti-patterns в E2E.
